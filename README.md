Julia versus OpenMP performance benchmarking project focused on scientific parallel computing and multithreading. The primary objective is to create a rigorous comparative analysis framework using PolyBench kernels implemented in Julia with multiple optimization strategies (sequential, SIMD, threaded, BLAS, tiled). The project targets deployment on the DAS-5 cluster at VU Amsterdam for high-performance computing research, with emphasis on measuring speed, time, speedup, efficiency, and scalability metrics.
The work follows scientific computing standards with careful attention to Julia's column-major memory layout, zero-allocation hot paths, and proper JIT warmup handling. SpawnAl has demonstrated deep expertise in HPC, cluster computing, and Julia programming, consistently requesting state-of-the-art implementations rather than basic guidance. The project structure is organized under "TheSpawnal/Julia_versus_OpenMP/julia_polybench" with modular architecture supporting individual benchmark launches, CSV-based results export, and flame graph profiling integration.
Current state
The benchmarking suite has evolved through multiple refactoring cycles and now includes a robust distributed computing extension using both MPI.jl and Distributed.jl for multi-node deployment. Recent work has focused on implementing sophisticated MPI strategies for kernels like Jacobi2D (with ghost cell exchange patterns), 3MM (triple matrix multiplication with various distribution approaches), Cholesky (block decomposition), and Nussinov (RNA folding with wavefront parallelism).
The core benchmarking framework uses manual time_ns() timing rather than BenchmarkTools for HPC deployment predictability, with four-phase timing pipeline including JIT warmup isolation, garbage collection control, and statistical measurement collection. Critical bugs in efficiency calculations have been resolved, ensuring parallel efficiency formulas only apply to appropriate strategies while distinguishing between parallel scaling and algorithmic improvements.
On the horizon
The distributed computing implementations are ready for comprehensive scaling studies across DAS-5's multi-node infrastructure, with SLURM scripts configured for both single-node (16 cores) and multi-node (up to 4 nodes, 64 cores) deployment. The project is positioned for extensive performance analysis comparing Julia's distributed computing capabilities against traditional OpenMP approaches, with sophisticated job scheduling and automated scaling studies planned.
Key learnings & principles
SpawnAl has established that proper efficiency calculations must distinguish between parallel strategies (where E_p = S_p / p applies) and non-parallel optimizations like BLAS (where efficiency metrics are not applicable). The project has proven that manual timing with explicit control over warmup, garbage collection, and state reset provides superior predictability for HPC deployment compared to standard benchmarking tools.
Critical insights include understanding DAS-5's hardware limitations (16 physical cores per node making 32-thread hyperthreading counterproductive for compute-bound workloads), the importance of InfiniBand-optimized MPI configurations, and the need for scale-aware verification tolerances that account for accumulated floating-point errors in large-scale computations.

Tools & resources
The project leverages Julia's ecosystem including MPI.jl and Distributed.jl for parallel computing, BLAS integration for optimized linear algebra, and Profile.jl with ProfileSVG for flame graph generation. Deployment targets the DAS-5 cluster system with SLURM job scheduling, utilizing InfiniBand interconnect for multi-node communication and CPU-only nodes for compute-intensive workloads.
